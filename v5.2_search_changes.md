## 二、models/backbone.py 改动

**v5.1.1 → v5.2：** `BackboneMobileNetV3` 参数化 `head_type` 和 `cut_point`

### 新增构造参数

```python
def __init__(
    self,
    num_classes: int = 200,
    pretrained:  bool = True,
    head_type:   str  = "full_network",   # 新增
    cut_point:   str  = "stage1",         # 新增（v5.1.1 硬编码为 stage1）
):
```

### cut_point 截断配置表

```python
CUT_CONFIG = {
    "stage1": (slice(0, 4), slice(4, None), 24, 16),  # (早期层, 晚期层, 通道数, 空间尺寸)
    "stage2": (slice(0, 7), slice(7, None), 40,  8),
}
```

| cut_point | 层范围 | 输出形状 | feat_channels |
|-----------|--------|----------|---------------|
| `stage1` | features[0:4] | (B, 24, 16, 16) | 24 |
| `stage2` | features[0:7] | (B, 40, 8, 8) | 40 |

### head_type 支持四种强度

| head_type | 结构 | 预期精度 |
|-----------|------|----------|
| `linear` | Linear(C, 200) | 25~35% |
| `mlp_small` | Linear(C,128)→ReLU→Linear(128,200) | 35~45% |
| `mlp_medium` | Linear(C,256)→ReLU→Dropout→Linear(256,200) | 40~50% |
| `full_network` | features_late→GAP→Linear(576,200) | 55~62% |

### 新增属性（供 moe_system 读取）

```python
self.feat_channels = feat_ch   # 24 or 40
self.feat_spatial  = feat_sp   # 16 or 8
```

## 三、models/moe_system.py 改动

**v5.1.1 → v5.2：** 支持从 `config dict` 一键构建，维度根据 `cut_point` 自动推导

### 新增 config dict 接口

```python
def __init__(
    self,
    config: dict = None,      # 新增：优先级高于关键字参数
    # ... 旧关键字参数保持向后兼容
):
```

使用示例：

```python
# v5.2 新接口（搜索时使用）
model = MoESystemTiny(config={
    "head_type":          "mlp_small",
    "cut_point":          "stage1",
    "num_experts":        4,
    "expert_hidden_dim":  96,
    "expert_num_layers":  2,
    "alpha_start":        0.7,
    "balance_loss_weight": 0.1,
})

# v5.1.1 旧接口（train.py 仍可使用）
model = MoESystemTiny(
    num_experts=4,
    compress_in=24,
    ...
)
```

### 维度自动推导规则

`config dict` 模式下，压缩器/解压器维度根据 `cut_point` 自动确定，无需手动传入：

```python
if config is not None:
    feat_ch        = self.backbone.feat_channels  # 从 backbone 读
    compress_in    = feat_ch          # 24 (stage1) or 40 (stage2)
    compress_out   = feat_ch          # 不压缩通道
    decompress_out = feat_ch * 2      # 48 (stage1) or 80 (stage2)
```

| cut_point | compress_in | compress_out | decompress_out |
|-----------|-------------|--------------|----------------|
| `stage1` | 24 | 24 | 48 |
| `stage2` | 40 | 40 | 80 |

## 四、search/search_space.py（新建）

定义所有可搜索变量和配置生成器。

### 搜索空间定义

```python
# Phase 1：核心变量（必搜）4×4×2 = 32 个组合
PHASE1_SPACE = {
    "head_type":  ["linear", "mlp_small", "mlp_medium", "full_network"],
    "alpha_end":  [0.3, 0.4, 0.5, 0.6],
    "cut_point":  ["stage1", "stage2"],
}

# Phase 2：次要变量（在 Phase 1 top-K 上搜）2×2×2×2 = 16 个组合
PHASE2_SPACE = {
    "expert_hidden_dim":   [64, 128],
    "num_experts":         [2, 4],
    "expert_num_layers":   [1, 2],
    "balance_loss_weight": [0.05, 0.1],
}

# Phase 1 搜索时固定的 Phase 2 默认值
PHASE2_DEFAULTS = {
    "expert_hidden_dim":   96,
    "num_experts":         4,
    "expert_num_layers":   2,
    "balance_loss_weight": 0.1,
}
```

### SearchConfig

```python
@dataclass
class SearchConfig:
    phase1_epochs:  int   = 10     # Phase 1/2 短训练轮数
    phase2_epochs:  int   = 10
    phase3_epochs:  int   = 100    # Phase 3 完整训练
    phase1_top_k:   int   = 3      # Phase 1 选出几个进 Phase 2
    phase3_top_k:   int   = 3      # 最终选几个做完整训练
    save_dir:       str   = "./search_results"
    alpha_start:    float = 0.7    # Stage 3 退火起点（固定）
```

## 五、search/search.py（新建）

三阶段搜索主脚本。

### 整体流程

```
Stage 2 共享训练
  ├── cut_point=stage1 → stage2_stage1.pt  (只跑一次)
  └── cut_point=stage2 → stage2_stage2.pt  (只跑一次)

Phase 1：搜核心变量
  ├── 32 个配置，每个 10 epoch Stage 3 短训练
  └── 保存 phase1_results.json，排序选 top-3

Phase 2：搜次要变量
  ├── 基于 Phase 1 top-3 各扩展 16 个组合 = 48 个配置
  ├── 每个 10 epoch Stage 3 短训练
  └── 保存 phase2_results.json，排序选 top-3

Phase 3：完整训练
  ├── Phase 2 全局 top-3，每个完整训练 100 epoch
  ├── 自动运行 eval_experts + eval_alpha_sweep
  └── 写 search_summary.txt
```

### 关键设计：Stage 2 共享

- Stage 2 只训练主干特征提取能力，**与 head_type 无关**
- 不同 `head_type` 共享同一个 `cut_point` 的 Stage 2 权重
- 因此只需按 `cut_point` 各跑一次（2 次），不是 32 次
- 加载时使用 `strict=False`，允许 head 结构不同

```python
# 2 次 Stage 2，不是 32 次
for cp in ["stage1", "stage2"]:
    run_stage2_once(cp, ...)   # 保存 stage2_{cp}.pt
```

### alpha 退火调度

```python
def compute_alpha(epoch, total_epochs, alpha_start, alpha_end, warmup=20):
    if epoch <= warmup:
        return alpha_start - (alpha_start - alpha_end) * (epoch / warmup)
    return alpha_end
```

短训练（10 epoch）时 warmup=20，实际上退火未完成，alpha 仍在 [alpha_start, alpha_end] 之间线性变化，能区分不同 alpha_end 配置的差异。

### 输出结构

```
search_results/
├── phase1_results.json          # 32 个配置的排名
├── phase2_results.json          # 48 个配置的排名
```

## 六、运行方法

```bash
# 全自动一键搜索（推荐，约 11h）
python search/search.py --phase all --gpus 0 > search_results/search.log 2>&1 &

# 分阶段运行
python search/search.py --phase 1 --gpus 0 > search_results/phase1.log 2>&1
python search/search.py --phase 2 --gpus 0 > search_results/phase2.log 2>&1
python search/search.py --phase 3 --gpus 0 > search_results/phase3.log 2>&1
```

**注意：** `search.py` 需要从项目根目录运行（`sys.path.insert(0, os.path.dirname(__file__))` 指向 `search/` 目录，但 `config`、`data`、`models` 等导入依赖项目根）。建议：

```bash
cd /path/to/cla_5
python search/search.py --phase all --gpus 0
```

## 八、与 v5.1.1 的主要区别

| 维度 | v5.1.1 | v5.2-search |
|------|--------|-------------|
| 参数选择 | 人工猜测 | Grid Search 自动搜索 |
| head_type | 固定 full_network | 搜索 4 种 |
| cut_point | 固定 stage1 | 搜索 stage1/stage2 |
| alpha_end | 固定 0.5 | 搜索 0.3/0.4/0.5/0.6 |
| num_experts | 固定 4 | 搜索 2/4 |
| expert_hidden | 固定 96 | 搜索 64/128 |
| 配置构建 | 硬编码关键字参数 | config dict 动态构建 |
| 维度 | 手动对齐 | 根据 cut_point 自动推导 |

## 九、已知限制

1. **search.py 导入路径**：`sys.path.insert(0, os.path.dirname(__file__))` 只加入了 `search/`，但 `config`、`data`、`models` 在项目根，需从根目录运行脚本。

2. **Phase 3 早停缺失**：`run_single_config` 跑 Phase 3 时没有早停（`train.py` 有，`search.py` 的 `run_single_config` 没有），100 epoch 全跑完。若需早停，需在 `run_single_config` 中加 `no_improve_count` 逻辑。

3. **`full_network` head 的 Stage 2 维度问题**：`full_network` head 期望 576 维输入（完整 MobileNetV3 输出），但 Stage 2 结束时保存的是 `full_network` 的 checkpoint，加载到其他 head_type（如 `mlp_small`）时 `strict=False` 会跳过 head 权重，只保留主干权重，符合预期。

4. **`GradScaler` 构造**：`search.py` 使用 `GradScaler('cuda', enabled=use_scaler)`，`train.py` 使用 `GradScaler(enabled=use_scaler)`。两者功能相同，注意 PyTorch 版本兼容性。
